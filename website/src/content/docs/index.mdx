<div class="feature-grid">
  <div class="feature">
    <div class="feature-head">
      <i class="ti ti-fingerprint feature-icon fi-orange"></i>
      <h3>RBAC + ABAC</h3>
    </div>
    <p>Hierarchical role <strong>inheritance</strong> with <strong>deny-overrides</strong>, combined with attribute-based rules — both classic models in one engine.</p>
  </div>
  <div class="feature">
    <div class="feature-head">
      <i class="ti ti-user-shield feature-icon fi-green"></i>
      <h3>Enforced Ownership & Groups</h3>
    </div>
    <p><code>own</code> verifies the record truly belongs to the requester (set <code>ownerField</code> / <code>owner</code> once). Grant to role <strong>groups</strong> & resource <strong>categories</strong> for bounded bulk access.</p>
  </div>
  <div class="feature">
    <div class="feature-head">
      <i class="ti ti-new-section feature-icon fi-purple"></i>
      <h3>Custom Actions & Gates</h3>
    </div>
    <p>Go beyond CRUD with <code>.action()</code> / <code>.do()</code>, and add mandatory <code>require()</code> gates that can only restrict access.</p>
  </div>
  <div class="feature">
    <div class="feature-head">
      <i class="ti ti-arrow-loop-right-2 feature-icon fi-blue"></i>
      <h3>Conditionals & Async Checks</h3>
    </div>
    <p>Attach conditions with <code>.where()</code> — comparisons, <code>in</code>, <code>cidr</code>, time windows (string or JSON). Resolve custom / async logic via <code>grantedAsync</code>.</p>
  </div>
  <div class="feature">
    <div class="feature-head">
      <i class="ti ti-shield-check feature-icon fi-red"></i>
      <h3>Hardened & Fail-Closed</h3>
    </div>
    <p><code>tryCan()</code> never throws; prototype-pollution-safe names; opt-in, ReDoS-guarded regex; redacted errors with <code>err.code</code>. 100% coverage, mutation-tested & fuzzed. — <a href="/accesscontrol/security/">Security</a></p>
  </div>
  <div class="feature">
    <div class="feature-head">
      <i class="ti ti-timeline-event-exclamation feature-icon fi-yellow"></i>
      <h3>Events & Audits</h3>
    </div>
    <p>An <code>access</code> event on every decision — granted <em>and</em> denied, with a <code>reason</code> — plus <code>change</code> and <code>error</code> hooks for a full audit trail.</p>
  </div>
</div>

```js
import { AccessControl } from 'accesscontrol';

const ac = new AccessControl({}, { policy: { ownerField: 'ownerId' } });

ac.grant('user').readOwn('profile', ['*', '!password']);
ac.grant('editor').where('$.article.status == draft').action('publish', 'article');

ac.can('user', { user: { id: 7 }, profile: { ownerId: 7 } }).readOwn('profile').granted; // true
```

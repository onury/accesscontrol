# AccessControl — documentation site

This folder is the documentation site for **AccessControl**, built with
[Astro](https://astro.build) + [Starlight](https://starlight.astro.build) and
themed by [`@onury/docs-kit`](https://github.com/onury/docs-kit). It deploys as a
GitHub Pages **project** site, served at **https://onury.io/accesscontrol**.

> Generated by `@onury/docs-kit`. The shared look (fonts, colors, header, code
> blocks, feature cards) lives in the kit; everything in this folder is yours to
> edit. Re-running `docs-kit init --force` regenerates the scaffold.

## Develop

```bash
npm --prefix website install
npm --prefix website run dev      # preview at localhost
npm --prefix website run build    # production build → website/dist
```

(If the kit is wired as a local `file:` dependency for theme development, install
with `npm --prefix website install --install-links`.)

## What comes from the kit vs. what's yours

| Shared (from `@onury/docs-kit`) | Project-owned (this folder) |
| --- | --- |
| Theme CSS (`styles/custom.css`, `theme.css`) | `src/styles/overrides.css` — your tokens (accent…) |
| Markdown-sync engine (`scripts/sync-docs.mjs` calls it) | `src/styles/hero.css` — the home hero look |
| TypeDoc / remark plumbing (inlined in `astro.config.mjs`) | `src/content/docs/**` — all pages |
| | `astro.config.mjs` — title, sidebar, options |

## Customize

### Accent color (and other theme tokens)

Give this site its own color in **`src/styles/overrides.css`** (loaded after the
shared theme, so it wins). One line is enough:

```css
:root {
  --oy-accent: #3b82f6;            /* this site's accent (links, primary button, active nav) */
  /* --oy-accent-high: var(--oy-accent); */
}
```

`--sl-color-accent` / `--sl-color-accent-high` derive from `--oy-accent`. For
finer control, override any Starlight `--sl-color-*` token in the same file —
see the [Starlight theming guide](https://starlight.astro.build/guides/css-and-tailwind/#styling-starlight).
Do **not** edit the kit's shared CSS for a single site; that changes every site.

### Home page & hero

- Content/hero copy: `src/content/docs/index.mdx` (frontmatter `hero:`).
- Hero background/styling: `src/styles/hero.css`.

### Pages, sidebar & navigation

- Add pages as Markdown/MDX under `src/content/docs/`.
- List them in the `sidebar` array in `astro.config.mjs`.

### Pulling in root docs (CHANGELOG, docs/*.md)

`scripts/sync-docs.mjs` copies repo-root markdown into the site at build time.
Edit its `files` list to add/remove sources; update the `sidebar` to match. The
synced files are generated (git-ignored) — edit the originals at the repo root.

### Component overrides (advanced)

Drop a Starlight override in `src/components/` and pass it via the `components`
option in `astro.config.mjs` (e.g. `components: { Hero: './src/components/Hero.astro' }`).

## Important: don't `import` the kit in `astro.config.mjs`

The theme is wired via **CSS string paths** (`'@onury/docs-kit/styles/*.css'`),
not a JS import. `@onury/docs-kit` is ESM-only, and importing it into the Astro
config makes Vite externalize `@astrojs/starlight` and load its TypeScript entry
under Node, which fails on Node ≥22.18. The `sync` engine is the one exception —
it's imported in `scripts/sync-docs.mjs`, which runs under plain Node.

## Deploy

`.github/workflows/docs.yml` builds this site and deploys it to GitHub Pages on
push to `master`. One-time: **Settings → Pages → Source = "GitHub
Actions"**. `base` in `astro.config.mjs` must stay `/accesscontrol` (it must match the
repo name, or assets 404).

## Update the shared theme

```bash
npm --prefix website update @onury/docs-kit
```

The kit is pinned to `github:onury/docs-kit#v1`; updating rolls in the latest shared theme.
